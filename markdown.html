A priori, o fator de inflação da variância (VIF) é uma maneira de medir a multicolinearidade, que avalia o quanto a variância de um coeficiente de regressão estimado aumenta se as suas preditoras estiverem correlacionadas. Se nenhum fator estiver correlacionado, os VIFs serão todos 1. Se o VIF for igual a 1 não há multicolinearidade entre os fatores, mas se o VIF for maior que 1, as preditoras podem estar moderadamente correlacionadas. A saída abaixo mostra que o VIF para os fatores de BtM, Ativo total e RPLP são menores que 1.5, o que indica alguma correlação, mas não o suficiente para se preocupar demais com isso. Isso porque um VIF entre 5 e 10 que indica alta correlação, o que pode ser problemático. E se o VIF for acima de 10, segundo Draper e Smith (1998), pode-se assumir que os coeficientes de regressão estão mal estimados devido à multicolinearidade. Logo, os demais fatores, ROA e ROE, estão entre 4,6 e 5, isto é, apresentam média-alta correlação.

O teste de Shapiro é utilizado para avaliar se a distribuição dos residos é semelhante a uma distribuição normal. Desse modo, a sua hipótese nula (H0) seria: a distribuição é semelhante a uma distribuição normal e a sua hipótese alternativa (H1) seria: a distribuição não é semelhante a uma distribuição normal. No caso estimado, como o P-valor do modelo é muito baixo, há uma chance muito pequena de se rejeitar a hipótese nula e estar errado, logo, aceitamos a hipótese nula proposta. O teste Jarque-Bera, por sua vez, é utilizado para testar a normalidade do modelo, ou seja, se o modelo pertence ou não à uma distribuição normal. Dessa forma, a hipótese nula (H0) seria: y1, y2, . . . , yn ∼ N(µ, σ2 ) enquanto a hipótese alternativa seria que o modelo não pertence à uma distribuição normal. No caso do nosso modelo testado, como o P-valor também é muito baixo, a hipótese nula não é rejeitada. O teste de Breusch-Pagan testa a heterocedasticidade do modelo. Ou seja, ele testa se o modelo apresenta variâncias do termo de perturbação não iguais para todas as observações. Desse modo, a hipótese nula (H0) será que as variâncias são diferentes nas observações e a hipótese alternativa (H1) será que as variâncias não são diferentes. Desse modo, como o P-valor é infinitamente baixo, a hipótese nula não é rejeitada. O teste de Durbin-Watson testa a presença de autocorrelação entre os erros do modelo. Ou seja, ele testa se os erros das observações são correlacionados ou não. Desse modo, a sua hipótese nula (H0) será que os erros são correlacionados enquanto a hipótese alternativa (H1) será que esses erros não são correlacionados. Pelo resultado do nosso teste, como o P-valor é alto (0,3524) nós rejeitamos a nossa hipótese nula e aceitamos a hipótese alternativa, em que os erros do modelo não estão correlacionados. ## Questão 8 Realize os testes de heterocedasticidade (White e Breusch-Pagan) para verificar a heterocedasticidade nos resíduos do modelo e analise o resultado dos mesmos.7

A priori, é importante diferenciar o conceito de homocedasticidade e heterocedasticidade. Desse modo, respectivamente, o primeiro conceito afirma que a variância dos erros E, condicionada aos valores das variáveis explanatórias, será constante. Em contrapartida, na heterocedasticidade, a variância dos erros será diferente para cada valor condicional de Xj. Assim, na presença de heterocedasticidade dos erros, os estimadores de MQO continuam sendo não viesados e consistentes, mas deixam de ser eficientes (ou seja, não possuem mais variância mínima). Outra importante consequência da heterocedasticidade é o viés do estimador da variância de beta, mesmo para amostras grandes (inconsistência). Como resultado, as estatísticas de teste t e F deixam de ser válidas, pois dependem da variância do estimador. Assim, duas formas de identificar a heterocedasticidade são os testes estatísticos: Teste de Breusch-Pagan e o Teste de White. Logo, analisando as saídas do R a seguir, nota-se que, em ambos os testes o p-valor é não significativo (<5%). Dessa forma, a probabilidade de erro ao rejeitar H0 é menor que 0,05. robustes <- function(data,reg, vector_area) {
  results <- matrix(,
    nrow = 10000,
    ncol = length(names(reg$coefficients)))
  name_coeficientes <- names(reg$coefficients)

  colnames(results) <- name_coeficientes
  for (i in 1:10000) {
    index_amostras <- sample(1:dim(data)[1],size= dim(data)[1], T)
    amostra <- data[index_amostras,]
    mod <- lm(log(Div) ~ BtM + RPLP + ROE + log(AtivoTotal), data = amostra)
    results[i,] <- mod$coefficients
  }
  par(mfrow = vector_area,
      mar = c(2, 2, 2, 2))
  name <- name_coeficientes[1]
  Hist <- hist(results[, name], plot = F, breaks = 100)
  plot(Hist, main = name, xlab = "", col = ifelse(Hist$breaks <= quantile(results[, name], 0.025), "red", ifelse(Hist$breaks >= quantile(results[, name], 0.975), "red", "white")),
       xlim = c(-9,-5))

  name <- name_coeficientes[2]
  Hist <- hist(results[, name], plot = F, breaks = 100)
  plot(Hist, main = name, xlab = "", col = ifelse(Hist$breaks <= quantile(results[, name], 0.025), "red", ifelse(Hist$breaks >= quantile(results[, name], 0.975), "red", "white")),
       xlim = c(-0.8,-0.2))

  name <- name_coeficientes[3]
  Hist <- hist(results[, name], plot = F, breaks = 100)
  plot(Hist, main = name, xlab = "", col = ifelse(Hist$breaks <= quantile(results[, name], 0.025), "red", ifelse(Hist$breaks >= quantile(results[, name], 0.975), "red", "white")),
       xlim = c(-1,2))

  name <- name_coeficientes[4]
  Hist <- hist(results[, name], plot = F, breaks = 100)
  plot(Hist, main = name, xlab = "", col = ifelse(Hist$breaks <= quantile(results[, name], 0.025), "red", ifelse(Hist$breaks >= quantile(results[, name], 0.975), "red", "white")),
       xlim = c(-0.6,0.6))

  name <- name_coeficientes[5]
  Hist <- hist(results[, name], plot = F, breaks = 100)
  plot(Hist, main = name, xlab = "", col = ifelse(Hist$breaks <= quantile(results[, name], 0.025), "red", ifelse(Hist$breaks >= quantile(results[, name], 0.975), "red", "white")),
       xlim = c(1,1.3))
}
